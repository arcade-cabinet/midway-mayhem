# Obstacle GLB Asset Mapping

instead of inline Three.js primitive geometry. Assets live at
`public/models/obstacles/`.

## Render Architecture

```
TrackContent.tsx
  switch (ob.kind)
    'cone' | 'oil' | 'barrier' | 'gate'  →  <StaticObstacle kind=... />
    'hammer'                              →  <ThemedHammer />
    'critter'                             →  <ThemedCritter />

src/render/obstacles/
  ThemedObstacle.tsx     — components (StaticObstacle, ThemedHammer, ThemedCritter)
  themedAssets.ts        — OBSTACLE_ASSETS map + ALL_OBSTACLE_PATHS preload list
  __tests__/
    themedAssets.test.ts — node unit tests (data integrity + file existence)
  ThemedObstacle.browser.test.tsx — real-GPU screenshot test per kind
  __baselines__/         — pinned PNGs (generated by first browser test run)
```

## How to Update a GLB

1. Copy the new file to `public/models/obstacles/<kind>.glb`.
2. Update `scale` / `yOffset` in `src/render/obstacles/themedAssets.ts` if needed.
3. Run `pnpm test:node` to verify the unit test still passes.
4. Run `pnpm test:browser` to capture new screenshots.
5. Copy new PNGs from `.test-screenshots/obstacles/` to `src/render/obstacles/__baselines__/`.
6. Commit both the GLB and updated baselines in the same PR.
